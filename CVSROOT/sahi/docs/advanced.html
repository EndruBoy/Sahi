Multi Threading

Record and playback of web pages spanning multiple domains.

Scenario:

  1. Navigate to www.google.com
  2. Search for "Sahi Automation"
  3. Click on the link which points to "sahi.sourceforge.net"
  4. Click on any link "Downloading Sahi" on "sahi.sourceforge.net" page.
What we have done is navigate from one domain to another domain (ie google.com to sourceforge.net)

Problem:

Sahi will not record step 4 by default.

Solution:

Set multidomains=true in {sahi_root}/config/sahi.properties

Effects:

Step 4 will be recorded correctly.

Side-effects:

After setting multidomains=true it will not be possible to have multithreaded playback.
Additionally, sahi works as if there is only one universal session. So it would not be possible to record or playback more than one script at a time.

Why:

Sahi manages its own sessions through cookies. When domains change, the cookie which had been set on the first domain will not be available on the second domain.

Sessions:

Sessions are needed for
  • tracking global variables,
  • counter index of the last played script index
  • the script and suite information etc.
Sessions are tracked using cookies in Sahi.

Also see Multidomains

Multi Threaded Playback

While playing back recorded scripts as part of continuous integration build, we can run more than one sahi test in parallel.
This helps in reducing build time.

Note that multithreaded playback will not work correctly if multidomains=true in sahi.properties.
